kafka集群的搭建

@aidodoo · Jun 2, 2019 · 1 min read

本系列文章主要阐述大数据计算平台相关框架的搭建，包括如下内容： 基础环境安装 zookeeper集群的搭建 kafka集群的搭建 hadoop集群的搭建 hbase集群的搭建 spark集群的搭建 本文主要讲解kafka集群的搭建步骤。 下载 kafka官网下载安装包kafka_2.11-2.2.0.tgz 安装 cd ~ tar -zxvf kafka_2.11-2.2.0.tgz mv kafka_2.11-2.2.0 kafka 配置更改 编辑kafka.server配置文件 vi /home/oracle/kafka/config/server.properties 不同机器配置的broker.id 、advertised.listeners不一样，根据对应的主机名配置。以服务器bigdata1，则其server.properties需更改的配置如下： broker.id=1 advertised.listeners=PLAINTEXT://bigdata1:9092 num.network.threads=3 num.io.threads=8 socket.send.buffer.bytes=102400 socket.receive.buffer.bytes=102400 socket.request.max.bytes=104857600 log.dirs=/home/oracle/kafka/kafka-logs num.partitions=1 num.recovery.threads.per.data.dir=1 offsets.topic.replication.factor=1 transaction.state.log.replication.factor=1 transaction.state.log.min.isr=1 log.retention.hours=8 log.segment.bytes=1073741824 log.retention.check.interval.ms=300000 zookeeper.connect=192.168.220.200:2181,192.168.220.201:2181,192.168.220.202:2181/kafka zookeeper.connection.timeout.ms=6000 group.initial.rebalance.delay.ms=0 其他配置项参考后续文章或者官方文档，本文只概述集群模式的安装。 建立kafka日志相关目录 mkdir -p /home/oracle/kafka/kafka-logs 复制kafka目录到其他服务器，并更改对应的broker.id 、advertised.listeners配置 scp -r /home/oracle/kafka oracle@bigdata2:/home/oracle scp -r /home/oracle/kafka oracle@bigdata3:/home/oracle 启动和验证 建立日志目录 mkdir -p /home/oracle/kafka/logs 依次启动集群中的kafka，执行脚本如下： cd /home/oracle/kafka/logs nohup /home/oracle/kafka/bin/kafka-server-start.
